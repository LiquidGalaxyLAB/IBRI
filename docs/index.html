Module Hierarchy

  • clients
    • clients.admin: This file contains a class definition to add the Clients model into the django administration.
    • clients.models: This file contains the Client model that is used to registered the clients user data.
    • clients.urls: This file contains the routing to server to the client the HTML
    • clients.views: This file contains the functions that to manage the clients data as login or show user data.
  • config
    • config.urls: This file contains the routing from config package
    • config.views: This file contains the views corresponding to the admin area.
  • drones
    • drones.admin: This file contains a class definition to add the Drone model into the Django administration.
    • drones.models: This file contains the Drone model that is used to registered the drones data.
  • ibri
    • ibri.settings: Django settings for IBRI GSoC16 project.
    • ibri.urls: This file contains the routing from main package
    • ibri.wsgi: WSGI config for IBRI project.
  • search
    • search.admin: This file contains the definition of search models to admin implenetation in the administration panel.
    • search.management
      • search.management.commands
        • search.management.commands.launch
    • search.models: This file contains the model to Search package including the Route, WayPoint and Mission model.
    • search.views: Search views contains all the functions that are necesary to use in the search application.
  • utils
    • utils.aes: This file contains some algorithms to encrypt and decrypt based on AES256.
    • utils.google: This file contain the function to create shortened URL using Google Shortener
    • utils.tsp: This file contain the functions to solve a TSP problem using A* algorithm
    • utils.weatherApi: This file is a bridge to use HTTPS in local server using HTTP connectin between the server and the OpenWeatherMap API.
